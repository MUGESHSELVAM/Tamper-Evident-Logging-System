ℹ️ Security & Features

Hash Chaining: Each log entry includes a SHA-256 hash of the previous log. Any tampering breaks the chain and is immediately detected during verification.

Core Security Measures:

  • SHA-256 cryptographic hash verification
  • Input validation and sanitization
  • Brute-force protection via rate limiting (5 attempts per 60 seconds)
  • IP address logging for audit trails
  • Real-time alert generation for critical events
  • Password-protected admin access

🔒 Admin Password Configuration

The admin password must be configured using the ADMIN_PASSWORD environment variable before running the application.

Example: export ADMIN_PASSWORD="your_secure_password"

Avoid hardcoding credentials to maintain secure deployment practices.
